# 検証: Hyperliquid funding rateの予測可能性（2026-07-11）

`docs/リサーチ-非価格シグナル-2026-07-11.md` の未解決の問い「文献（Binance/Bybit・BTC/ETH中心）の実証がHyperliquid・SOLでも成立するか」の自前OOS検証。

- **データ**: Hyperliquid info API `fundingHistory`（無料・認証不要）。BTC/SOL各17,279時間（約23.7ヶ月、2024-07-21〜2026-07-11）
- **スクリプト**: `scripts/funding_analysis.py`（再実行可能）
- **手法**: ACF／符号持続／ウォークフォワードOOS 1期先予測（後半30%テスト・週次refit）／水準バケット→次24h累積funding

## 結論（3行）

1. **系列相関・持続性は文献どおり強く再現**（ACF lag1: BTC 0.86 / SOL 0.78、符号持続 0.94 / 0.91）→ fundingフィルターの前提は成立
2. **ただしAR型モデルは無変化モデルにほぼ勝てない**（文献のDAR優位は再現弱）→ 凝った予測モデルは不要。**「今の水準」をそのまま使う単純な閾値フィルターが正解**
3. **水準バケット→次24h累積fundingは綺麗に単調**で、極端域（上位10%）ではロング保有コストがBTC +0.087%/日・SOL +0.104%/日 ≒ taker手数料の約2倍/日 → 経済的に意味のある差

## 解釈と設計への示唆

1. **文献の再現状況**: 系列相関・持続性（CMU論文の核心）は完全に再現。一方、DAR型予測がランダムウォークに勝つという結果（SSRN 5576424）はBTCのAR(24)で+2.7%と僅かに再現、SOLでは再現せず。**予測モデルを作る価値は薄く、現在値の閾値判定で十分**
2. **フィルターの経済的意味**: 極端域でのロングは1日あたり約9〜10bpsのfundingコスト（taker往復手数料と同オーダー/日）。数日保有するトレンド戦略では**期待エッジを食い潰す規模**であり、逆にショートには同額の追い風
3. **SOLの独自性**: 負のfunding局面が多く、「funding<0のときのロング」は次24hでキャリー収入（+0.02%/日 ≒ 年率7%相当の追い風）になる。BTCとSOLで同じ閾値を使わず銘柄別に設定すべき
4. **注意（誠実に）**: 本検証が示したのは「**fundingがfunding自身を予測する**」（＝保有コスト/収入が事前に読める）ことであり、「**fundingが価格を予測する**」ことではない。価格予測力（高funding→クラッシュ）は文献（BIS・月次ホライズン）由来の別レイヤーの話
5. **アンカー張り付き**: 半分前後の時間はアンカー値ちょうど＝情報なし。フィルターが動作するのは残り約半分の時間帯のみ

## 推奨する次のステップ

- **FundingGate（フィルター層）の実装**: 新戦略は作らず、既存戦略のエントリー時に適用
  - BTC/SOL共通: funding ≥ p90（直近ローリングで算出）→ 新規ロング抑制（サイズ半減 or 禁止）、ショートは通常どおり
  - SOL: funding < 0 → ロングのキャリー追い風としてそのまま通す（ボーナス扱いはしない）
  - パラメータ（p90/固定閾値、抑制の強さ）は**PBO/DSRゲートを通して選ぶ**（多重検定を増やさないため候補は2〜3個に固定）
- ドライランで「ゲート発動時間帯・回避できたfundingコスト」をログし、実測で効果を確認してから本採用
